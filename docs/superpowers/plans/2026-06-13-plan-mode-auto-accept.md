# Implementation Plan: Auto Accept and Plan Mode

**Status:** Completed

## Overview
This document outlines the design and implementation of two new project-wide execution modes: **Auto Accept** and **Plan Mode**. These features provide granular control over the autonomous agent's execution pipeline, addressing the user's need for automated headless execution as well as cautious preview-driven execution.

### Goals
1. **Auto Accept**: Bypass all user confirmation loops for tool executions (previously partially supported via `--yolo` flag, now fully integrated into the project-wide configuration).
2. **Plan Mode**: Enforce a strict confirmation loop where *all* tool calls generated by the LLM are presented to the user for preview and manual approval before any execution occurs.

## Design

### 1. Configuration Changes
Both features are configurable globally via `.deepcode/settings.json` or locally through CLI arguments:
- Add `autoAccept` (boolean) to `DeepcodingSettings` and `ResolvedDeepcodingSettings`.
- Add `planMode` (boolean) to `DeepcodingSettings` and `ResolvedDeepcodingSettings`.
- Expose these via system environment variables (`DEEPCODE_AUTO_ACCEPT`, `DEEPCODE_PLAN_MODE`).

### 2. CLI Updates
- Update `src/cli.tsx` to accept a `--plan` flag which sets `planModeEnabled = true`.
- Continue mapping `--yolo` and `-y` flags to `autoAcceptEnabled = true`.
- Pass these toggles down to `<AppContainer />` and the `App` props.

### 3. Session Manager Enhancements
- Expand `SessionManagerOptions` in `src/session.ts` to accept `planMode?: boolean`.
- Within the `SessionManager`'s constructor, bind `this.planMode = options.planMode ?? false`.
- During `this.handleAssistantMessage` processing, when computing tool call permissions, pass `planMode: this.planMode` into `computeToolCallPermissions()`.

### 4. Permission System Upgrades
- Update `ComputeToolCallPermissionsOptions` in `src/common/permissions.ts` to accept an optional `planMode?: boolean`.
- **Implementation Logic**:
  - If `autoAccept` is true, all tool calls map to `permission: "allow"`.
  - If `planMode` is true (and `autoAccept` is false), bypass the default scope risk evaluation. Automatically map *every* tool call to `permission: "ask"`.
  - Add all tool calls to `askPermissions` so they are fully displayed in the `PermissionPrompt` UI.
  - If neither is true, default to the existing scope evaluation logic.

## Verification
- Running `deepcode --plan` forces the Agent to pause and display the "Permission required" prompt for every tool (including benign ones like `read`), effectively creating a plan review sequence.
- Running `deepcode --yolo` (Auto Accept) bypasses all execution checks cleanly.
- Updating `~/.deepcode/settings.json` with `"planMode": true` achieves project-wide plan-driven behavior without needing CLI flags.
